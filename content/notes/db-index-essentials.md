## 索引是什么

索引就像书的**目录**——用空间换时间，用额外的存储结构加速查询。

MySQL InnoDB 引擎使用 **B+ 树**作为索引结构：

- **非叶子节点**只存键值（路由）
- **叶子节点**存完整数据（聚簇索引）或主键引用（二级索引）
- 叶子节点之间用**双向链表**连接，范围查询高效

## 聚簇索引 vs 二级索引

| 类型 | 存储内容 | 对应数据 |
|------|----------|----------|
| 聚簇索引（主键） | 叶子节点 = 完整行数据 | 直接定位 |
| 二级索引 | 叶子节点 = 索引列 + 主键 | 需要**回表**查主键索引 |

> 回表：通过二级索引找到主键 → 再到聚簇索引找到完整行。覆盖索引可以避免回表。

## 最左前缀原则

联合索引 `(a, b, c)` 实际上建立了：

- `a` 的索引
- `(a, b)` 的索引
- `(a, b, c)` 的索引

```sql
-- ✅ 能命中索引
WHERE a = 1
WHERE a = 1 AND b = 2
WHERE a = 1 AND b = 2 AND c = 3

-- ❌ 不能命中索引
WHERE b = 2
WHERE c = 3
```

## 实践要点

1. 区分度高的列放前面
2. 避免 `SELECT *`，尽量使用覆盖索引
3. `EXPLAIN` 看执行计划，关注 `key` 和 `rows`
4. 索引不是越多越好——写入有维护成本
