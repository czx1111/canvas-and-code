## 根本机制：CPL 寄存器

内核态与用户态的区分，根本上取决于 CPU 的 **CPL（Current Privilege Level，当前特权级）** 寄存器：

| CPL 值 | 名称 | 简称 |
|--------|------|------|
| 0 | 内核态 | Ring 0 |
| 3 | 用户态 | Ring 3 |

> CPL 是 CPU 硬件层面的特权标志，不是操作系统"发明"的概念，而是 x86 架构原生提供的保护机制。

## 切换入口：三种触发方式

从用户态进入内核态，只有三条路径：

```
外部中断（硬件）    ← 网卡、键盘、时钟等设备发出
内部中断（异常）    ← 除零、缺页、非法指令等 CPU 自动触发
系统调用（主动）    ← 用户程序主动请求内核服务
```

| 切换方式 | 触发主体 | 典型场景 |
|----------|----------|----------|
| 外部中断 | 硬件设备 | 网卡收到数据包、定时器到期 |
| 内部中断 | CPU 自身 | 缺页异常、除零错误 |
| 系统调用 | 用户程序 | `read()`、`write()`、`fork()` |

> 无论哪条路径，本质都是 CPU 从 CPL=3 切换到 CPL=0。

## 执行环境变化：CPU 三大动作

切换发生的瞬间，CPU 硬件自动完成三大动作：

1. **CPL 改为 0** — 特权级提升到内核态
2. **PC（RIP）跳转** — 指令指针指向内核空间的入口地址
3. **RSP 切换为内核栈** — 栈指针切换到当前进程的内核栈

| 寄存器 | 用户态 (CPL=3) | —切换→ | 内核态 (CPL=0) |
|--------|---------------|--------|----------------|
| RIP    | 用户代码地址    | →      | 内核入口地址     |
| RSP    | 用户栈          | →      | 内核栈          |
| CPL    | 3              | →      | 0              |

> 栈切换是安全的关键：如果继续用用户栈，用户程序可以篡改栈内容来攻击内核。

## 权限差异：内核态解锁了什么

进入内核态后，CPU 解锁了三大权限：

| 权限 | 内核态 (Ring 0) | 用户态 (Ring 3) |
|------|-----------------|-----------------|
| 特权指令 | ✅ `hlt`、`cli`、`sti`、`lidt` 等 | ❌ 执行即触发异常 |
| 硬件 I/O 端口 | ✅ 直接读写 `in`/`out` | ❌ 需经内核驱动 |
| 物理内存访问 | ✅ 可访问全部物理内存 | ❌ 仅限自身虚拟地址空间 |

- **特权指令**：如关中断（`cli`）、停机（`hlt`）、加载中断描述符表（`lidt`）等，只有内核态才能执行
- **I/O 端口**：内核态可以直接操作硬件端口，用户态必须通过系统调用委托内核完成
- **物理内存访问**：内核态拥有全物理内存的访问权限，用户态只能访问分配给自己的虚拟地址空间

> 三大权限合在一起，意味着内核态拥有对系统和硬件的**绝对控制权**。

## 总结

CPL 寄存器（决定当前态）→ 三种入口（触发切换）→ 三大动作（CPL→0, RIP→内核, RSP→内核栈）→ 三大权限（特权指令 + I/O + 全物理内存）

内核态与用户态的本质就是 CPU 硬件提供的**特权分级保护机制**。操作系统利用这一机制，将关键资源操作限制在内核态，从而隔离用户程序，保证系统安全和稳定。
