## 内核空间与用户空间

Linux 将虚拟地址空间划分为**内核空间**与**用户空间**。在现代 64 位系统中，这两者的边界由 `PAGE_OFFSET` 定义。

## 直接映射（线性映射）

系统启动时，引导程序将内核核心基础映像（`vmlinux`）加载至物理内存。与此同时，操作系统会为**整个物理内存**（而非仅仅内核映像）建立一个到内核虚拟地址空间的**直接映射**（或称线性映射）。

### 本质：变址寻址

该映射机制的本质类似于**变址寻址**，即确立一个固定的数学关系：

```
虚拟地址 = 物理地址 + PAGE_OFFSET
物理地址 = 虚拟地址 - PAGE_OFFSET
```

内核中通过 `__pa()` / `__va()` 两个宏完成这一转换：

```c
// 虚拟地址 → 物理地址
#define __pa(x)     ((unsigned long)(x) - PAGE_OFFSET)

// 物理地址 → 虚拟地址
#define __va(x)     ((void *)((unsigned long)(x) + PAGE_OFFSET))
```

### 为什么高效

得益于直接映射，当内核访问直接映射区的虚拟地址时：

- **软件层面**：无需遍历复杂的 VMA 链表或动态建立页表，仅通过一次简单的**整数加减运算**即可计算出对应的物理地址
- **硬件层面**：仍依赖 MMU 与 TLB 完成最终的地址转换，但由于页表已预置且映射连续，TLB 命中率极高，转换效率极高

## 对比：用户空间的地址转换

| 对比项 | 内核直接映射区 | 用户空间 |
|--------|---------------|----------|
| 地址转换 | `__pa()` / `__va()` 整数运算 | 遍历页表（MMU 硬件完成） |
| 页表是否预置 | 是（启动时建立） | 否（按需建立，触发缺页中断） |
| 映射是否连续 | 是（线性连续） | 不一定（按需分配，可能碎片化） |
| 性能 | O(1)，极快 | 依赖 TLB 缓存，未命中时开销大 |

## 总结

> 直接映射是现代虚拟内存系统下内核高效运转的重要基石。

它的核心思想很简洁：**用一段固定的偏移量，把整个物理内存"平移"到内核虚拟地址空间**。这样内核访问物理内存时，就不需要走完整的页表查找流程，一个加减法就够了。

但需要注意，直接映射区映射的是**全部物理内存**，而不仅仅是内核映像本身。这意味着内核可以通过 `__va()` 访问任意物理页——这对内核管理物理内存是必要的。
